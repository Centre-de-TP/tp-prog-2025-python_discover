The "Variables"

Variables are a way to keep thing in memory for a program.
Imagine a simple program where you ask the user which is his favourite music so you can launch it for him.
Then a possible way of using variables could be to store the user response into one variable and once you need to launch the music, you take out the variable to search for the correct music on internet.

A variable can be seen as a box where you can put every kind of things and just re-use these things when you need to.
To find these boxes in the big box storage which is your computer, you need to put a tag on the box.
It's called the variable name.

You can see an example below :

favouriteMusic = input("What's your favorite music?") print(favouriteMusic) >>> Believers from Imagine Dragons

I hope you remember the input function we saw earlier.
Now we added a variable to store the user response.
Then the variable can be re-used with the print function so you can display the result.

This example showed us the way of storing a user input but you can also store objects in a variable.
Let's say that we want to store the artist name directly into the code without asking the user :

favouriteAuthor = "Imagine Dragons" print(favouriteAuthor) >>> Imagine Dragons

To go a bit further with variables, it's possible to variable, which store copies of other variables.

favouriteAuthor = "Imagine Dragons" copy = favouriteAuthor print(favouriteAuthor) print(copy) >>> Imagine Dragons Imagine Dragons

As said previously, a variable can store any kind of object in python.
We saw how to store sentences but let's see if I want to store a number :

famousNumber = "42" print("The answer of life is ", famousNumber) >>> The answer of life is 42

The "Comments"

Another useful thing to structure code is the comments.
They are represented by the symbole "#" and the purpose of the comments are just to explain the user what is happening.

#This line isn't useful at all famousNumber = 42 print("The answer of life is ", famousNumber) >>> The answer of life is 42

As you could see previously, the comment didn't change the code behaviour at all.

The "Types"

Let's now introduce the object type.
In python, and in programmation in general we often manipulate object.
But each object has his own type, and you can have nested types in some programming languages.

For example, since the beginning of the tutorial you are using sentences which in programming are called "string".
A lot of other types exists but for now let's concentrate on the principals ones.

# Primordial types print(type(42)) #The integer number are called int. >>> int print(type(42.5)) #The number with a comma (or in programming with a dot) are called float. >>> float print(type("text")) #The texts are called string (written str). >>> str print(type("42.5")) #The double quote are used to create a text. >>> str

As you could see, a function "type" can be used to see the object type.
Regarding the variables, their type is the type of the held object.
If the held object change, the variable type also change.

my_variable = 42 print(type(my_variable)) >>> int my_variable = 42 my_variable = 42.5 print(type(my_variable)) >>> float
